# America — одностраничник с чатом

Лендинг «Америка и почему она лучшая страна» на React + Tailwind. Регистрация/вход и общий чат в реальном времени. Реальный трафик и живое общение.

---

## Залить проект на GitHub (для Railway)

Если репозиторий уже создан на GitHub (например «Америка-посадка» / america-landing), выполни в терминале **в папке проекта** (где лежат `package.json` и папка `server`):

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/superpuperpubger-png/america-landing..git
git push -u origin main
```

(Подставь свой URL репозитория, если он другой — скопируй его с страницы «Быстрая настройка» на GitHub.)  
После этого в Railway нажми **Refresh** и выбери этот репозиторий.

## С телефона (одна сеть Wi‑Fi)

Чтобы зайти на сайт с телефона, компьютер и телефон должны быть в одной Wi‑Fi сети.

1. **Запусти на компьютере** (два окна терминала):

   Терминал 1 — бэкенд:
   ```bash
   cd server
   npm install
   npm start
   ```

   Терминал 2 — фронтенд:
   ```bash
   npm install
   npm run dev
   ```

2. **Узнай IP компьютера** (в терминале на ПК):
   - Windows: `ipconfig` → ищи «IPv4-адрес» (например `192.168.1.105`)
   - Mac/Linux: `ifconfig` или `ip addr` → адрес вида `192.168.x.x`

3. **На телефоне** открой в браузере:
   ```
   http://192.168.x.x:5173
   ```
   (подставь свой IP вместо `192.168.x.x`)

Трафик и чат будут общими: кто заходит с телефона или с ПК — все видят одни счётчики и один чат.

## Публичная ссылка (реальный трафик и чат для всех)

Чтобы заходили любые люди из интернета (в том числе с телефона) и был один общий чат и один счётчик визитов — нужно выложить проект в сеть.

### 1. Деплой бэкенда (Render.com, бесплатно)

1. Зайди на [render.com](https://render.com), зарегистрируйся.
2. **New → Web Service**.
3. Подключи репозиторий с проектом или залей папку `server` (Root Directory: `server`).
4. Настрой:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
5. В **Environment** добавь переменную `JWT_SECRET` (любая длинная случайная строка).
6. Нажми **Create**. Дождись деплоя и скопируй ссылку на сервис, например:
   ```
   https://america-chat-server.onrender.com
   ```

### 2. Деплой фронтенда (Vercel, бесплатно)

1. Зайди на [vercel.com](https://vercel.com), зарегистрируйся.
2. **Add New → Project** → импортируй репозиторий или залей папку проекта (корень — где лежит `package.json`, не `server`).
3. В настройках проекта **Environment Variables** добавь:
   - **Name:** `VITE_API_URL`
   - **Value:** твоя ссылка бэкенда **без** слеша в конце, например `https://america-chat-server.onrender.com`
4. Запусти деплой (Deploy). Vercel выдаст ссылку на сайт, например:
   ```
   https://america-landing.vercel.app
   ```

Эту ссылку можно скидывать кому угодно — с телефона и с ПК. Все будут в одном чате, трафик (визиты и онлайн) общий и реальный.

**Важно:** на бесплатном Render сервер «засыпает» после ~15 минут без заходов; первый заход после этого может открываться 30–50 секунд.

## Если Render.com не открывается

### Не открывается сам сайт Render.com

- **Блокировка / регион:** иногда Render бывает недоступен из некоторых стран или провайдеров. Попробуй:
  - другой браузер или режим «Инкогнито»;
  - VPN (например Cloudflare WARP, Proton VPN);
  - мобильный интернет с телефона вместо домашнего Wi‑Fi.
- **Временный сбой:** зайди позже или проверь [status.render.com](https://status.render.com).

### Не открывается твоё приложение на Render (ссылка типа xxx.onrender.com)

- **Долгая загрузка:** на бесплатном тарифе сервер «засыпает». Первый запрос после простоя может идти 30–60 секунд — подожди и обнови страницу.
- **Ошибка при открытии:** в браузере бэкенд отдаёт только API (JSON), а не страницу. Проверь так: открой в браузере `https://твой-сервис.onrender.com/api/traffic` — должен вернуться JSON с числами (totalVisits, onlineNow). Если видишь JSON — бэкенд работает, проблема может быть во фронтенде или в переменной `VITE_API_URL`.

### Альтернативы Render (бесплатный бэкенд)

Если Render не подходит, можно задеплоить тот же бэкенд (папка `server`) на другом хостинге:

| Сервис | Что делать |
|--------|------------|
| **Railway** | [railway.app](https://railway.app) → New Project → Deploy from GitHub или залить папку `server`. Укажи Start: `npm start`, порт подхватится из `PORT`. Скопируй URL сервиса и подставь его в `VITE_API_URL` на Vercel. |
| **Fly.io** | [fly.io](https://fly.io) → в папке `server`: `fly launch` → затем `fly deploy`. Дам URL вида `https://xxx.fly.dev` — его в `VITE_API_URL`. |
| **ngrok (без деплоя)** | Запусти бэкенд локально (`npm start` в `server`). В отдельном терминале: `ngrok http 3001`. ngrok выдаст ссылку вида `https://xxxx.ngrok.io` — её можно временно указать как `VITE_API_URL` и дать людям ссылку на фронт (если фронт тоже где-то открыт). Пока твой ПК и ngrok включены — чат и трафик будут работать. |

После смены бэкенда просто обнови в Vercel переменную **`VITE_API_URL`** на новый URL и пересобери проект (Redeploy).

## Стек

- **Frontend:** React, Tailwind CSS, Lucide React, Socket.IO Client  
- **Backend:** Express, Socket.IO, JWT, bcryptjs; данные пока в памяти (для постоянного хранения можно подключить БД).

Для продакшена задай `JWT_SECRET` в переменных окружения и при желании замени in-memory хранилище на БД (например PostgreSQL + Prisma).
