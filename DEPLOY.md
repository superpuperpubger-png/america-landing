# Как выложить сайт в интернет — одна ссылка для всех

Чтобы любой человек с любого устройства и из любой страны мог зайти на сайт по одной ссылке, нужно выложить **две части**: бэкенд (сервер) и фронтенд (сайт). Пользователи заходят только по ссылке на фронтенд — она и будет «одной ссылкой».

---

## Шаг 1. Код в GitHub

1. Зарегистрируйся на [github.com](https://github.com), если ещё нет аккаунта.
2. Создай новый репозиторий (например, `america-landing`).
3. В папке проекта выполни в терминале:

```bash
git add .
git commit -m "Prepare for deploy"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/america-landing.git
git push -u origin main
```

Подставь свой логин и имя репозитория вместо `ТВОЙ_ЛОГИН/america-landing`.

---

## Шаг 2. Деплой бэкенда (Render.com)

Бэкенд — это чат, авторизация и счётчик посещений. Он должен работать 24/7 в интернете.

1. Зайди на [render.com](https://render.com) и войди через GitHub.
2. **Dashboard** → **New** → **Web Service**.
3. Подключи репозиторий с этим проектом (если ещё не подключён).
4. Настрой сервис:
   - **Name:** `america-chat-server` (или любое).
   - **Region:** выбери ближайший к целевой аудитории (например, Frankfurt).
   - **Root Directory:** укажи `server` (обязательно).
   - **Runtime:** Node.
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
5. В **Environment** добавь переменную:
   - **JWT_SECRET** — нажми **Generate** (или придумай длинный случайный пароль).
6. Нажми **Create Web Service**. Дождись первого деплоя.
7. Скопируй **URL сервиса**, например: `https://america-chat-server.onrender.com` — это URL бэкенда. Он понадобится на следующем шаге.

Важно: на бесплатном плане Render «засыпает» после ~15 минут без заходов. Первый заход после этого может занять 30–50 секунд — это нормально.

---

## Шаг 3. Деплой фронтенда (Vercel)

Фронтенд — это то, что видят пользователи: страница, кнопки, чат. Одна ссылка будет вести сюда.

1. Зайди на [vercel.com](https://vercel.com) и войди через GitHub.
2. **Add New** → **Project** → выбери тот же репозиторий.
3. Настрой проект:
   - **Framework Preset:** Vite (должно определиться само).
   - **Root Directory:** оставь пустым (корень репозитория).
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. В **Environment Variables** добавь:
   - **Name:** `VITE_API_URL`
   - **Value:** URL бэкенда **без слеша в конце**, например: `https://america-chat-server.onrender.com`
5. Нажми **Deploy**. Дождись окончания сборки.
6. Vercel выдаст ссылку вида: `https://america-landing-xxx.vercel.app` — **это и есть твоя одна ссылка для всех**.

Дай эту ссылку пользователям — они смогут заходить с телефона, планшета или компьютера из любой страны.

---

## Проверка

1. Открой ссылку Vercel в браузере (в том числе с телефона).
2. Зарегистрируйся, зайди в чат, напиши сообщение.
3. Открой счётчик посещений — должны обновляться визиты и «онлайн сейчас».

Если чат не подключается или «нет связи» — проверь, что в Vercel в переменных окружения указан именно URL бэкенда (Render) без опечаток и без слеша в конце.

---

## Кратко

| Что            | Где деплоить | Результат                          |
|----------------|--------------|------------------------------------|
| Бэкенд (чат, API) | Render.com   | URL типа `https://xxx.onrender.com` |
| Фронтенд (сайт)   | Vercel       | **Одна ссылка** типа `https://xxx.vercel.app` |

Пользователи заходят только по ссылке Vercel — с неё всё работает: регистрация, вход, чат, трафик.
